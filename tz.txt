1. Мета роботи

Розробити систему логування для програм на C++, яка забезпечує збір, збереження та обробку логів з використанням стандартних контейнерів і алгоритмів STL.
Система повинна:

фіксувати події програми різного рівня важливості;
забезпечувати зручний доступ до логів та їхню обробку;
інтегруватися з будь-яким C++ проєктом;
дозволяти аналіз та сортування логів.
2. Джерела логів

Логи генеруються безпосередньо з програми під час її виконання:

Події у власному коді: інформаційні повідомлення, помилки, попередження, налагоджувальні дані.
Тестові або штучні події для демонстрації роботи системи.
Опціонально: події сторонніх модулів або API (якщо вони підключені до програми).
3. Основні функції системи
Збір логів
Рівні: DEBUG, INFO, WARNING, ERROR;
Фіксація часу та джерела події (клас або модуль).
Збереження логів
Використання STL-контейнерів: vector, list, deque, map;
Збереження у файл (TXT, CSV, JSON);
Можливість ротації файлів логів за розміром або датою.
Обробка логів
Фільтрація за рівнем важливості;
Сортування за часом або джерелом події;
Формування простих статистичних звітів (кількість помилок, попереджень).
Конфігурація
Налаштування рівнів логів та формату збереження;
Можливість додавання нових джерел логів;
Легка інтеграція з іншими програмними модулями.
4. Архітектура системи
Logger Core – збір та первинна обробка логів.
Log Storage – збереження логів у пам’яті та на файлах.
Log Formatter – форматування та структуризація логів.
Log Analyzer – фільтрація, сортування, статистика.
Configuration Module – налаштування рівнів та форматів логів.
Interface Module – API для підключення до інших модулів.
5. Приклад роботи системи
Генерація логів у коді програми:
Logger::log(INFO, "Початок обробки даних");
Logger::log(DEBUG, "Значення змінної x = 42");
Logger::log(WARNING, "Виявлено невідповідність формату");
Logger::log(ERROR, "Не вдалося відкрити файл config.txt");
Збереження логів у STL-контейнери:
vector для збереження послідовності подій;
map для групування логів за модулем або рівнем.
Збереження на диск у форматі CSV або JSON для подальшого аналізу.
Використання фільтрів та сортування для отримання статистики:
наприклад, всі помилки за останню годину;
кількість попереджень по кожному модулю.
6. Нефункціональні вимоги
Мінімальний вплив на виконання основної програми;
Надійність: логування не блокує роботу програми;
Кросплатформеність: Windows і Linux;
Зручність і зрозумілість коду;
Документованість і наочна демонстрація через тестові події.
7. Технології
Мова: C++17 або C++20;
STL: vector, list, deque, map, алгоритми сортування та пошуку;
Збереження: TXT, CSV, JSON;
IDE: Visual Studio, CLion, VS Code.
